# Separate run operator UI from Published Wiki HTML visualization

Operator observation of a Wiki Run stays CLI-native: JSON `wiki-run` for CI and an optional line-oriented TUI for interactive status; it is not replaced by a product web application or SPA run dashboard. Human browsing of a finished Wiki—including HTML rendering and a link-graph view derived from page cross-links—is a separate, optional, deterministically generated artifact of the Published Wiki (static HTML and optional graph data), not part of the Semantic Workflow, Staging Wiki, or Host Instructions. Mermaid and similar diagrams remain page content rendered by the visualization layer; the Host does not maintain a knowledge graph, and Wiki pages still must not embed raw HTML for publication. A browser-based live run console is deferred until the line-oriented TUI is proven insufficient, and even then it remains distinct from Wiki visualization.
