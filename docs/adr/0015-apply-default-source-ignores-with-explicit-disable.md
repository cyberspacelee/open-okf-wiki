# Apply Default Source Ignores with an explicit per-repository disable

A Repository Snapshot remains the pinned commit's tracked tree minus path exclusions, but the product now ships **Default Source Ignores** (noise such as dependency and build output trees) that apply unless the repository sets an explicit disable. User `ignore` patterns are always additive; writing a custom ignore never turns defaults off. There is no re-include/`!` negation in this step—paths that must remain evidence require disabling defaults for that repository and listing the desired exclusions explicitly. Provenance and the Wiki Run Record store both the disable flag and the **expanded Effective Source Ignores** frozen at run start so Manual Retry and publication metadata reproduce the same Snapshot membership without re-resolving a later product default catalog. Default Source Ignores are Host-owned platform policy, not Producer Skill content, and this decision does not adopt gitignore semantics or exclude tests by default.
