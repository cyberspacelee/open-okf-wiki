# Refresh

Treat the existing Staging Wiki as useful context, not as authority.

1. Reinspect the complete Repository Snapshot and reconsider every existing page against current
   source.
2. Preserve structure and prose that remain useful and accurate. Correct stale facts and citations;
   add pages for newly important reader questions; remove or merge pages whose purpose disappeared.
3. Repair navigation after every page-set change and ensure no old statement survives merely because
   its source file was untouched.
4. Proceed to the review reference after the whole Wiki reflects the current snapshot.

Refresh is complete when every retained page still earns its place, all current important ideas are
navigable, and no content depends on the previous publication being correct.
