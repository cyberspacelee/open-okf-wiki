# 03 — Derive Coverage Obligations from Markdown

**What to build:** Markdown requirements repositories produce auditable Major and Supporting Obligations, allowing users to see exactly which requirements and knowledge units are covered, excluded, deferred, or blocking publication.

**Blocked by:** 02 — Build a multi-repository Source Set.

**Status:** ready-for-agent

- [ ] Markdown headings and sections become stable Source Units.
- [ ] Numbered requirements, acceptance criteria, normative statements, tables, and glossary definitions produce Coverage Obligations.
- [ ] The Producer Profile classifies Obligations as Major or Supporting.
- [ ] Explicit exclusions and Supporting deferrals require recorded reasons.
- [ ] Open, blocked, or failed Major Obligations prevent publication.
- [ ] The coverage report shows totals and dispositions by source, role, and priority.
- [ ] Repeating inventory at the same Source Set produces stable Source Unit and Obligation identities.
- [ ] End-to-end tests demonstrate both a blocked run and a fully closed run.
