# Refresh

Treat the existing Staging Wiki as useful context, not as authority.

**Hard cut on page format:** do **not** preserve pre-OKF structure against the current contract.
Title-only frontmatter, overview narrative living in `index.md`, inline body `repo:` / `[Source](repo:…)`
citations, or agent-authored reserved docs must be rewritten to OKF concept pages (`overview.md` +
four-field frontmatter + `# Citations`) or removed. When the retained tree cannot be made compliant
without a full rewrite, regenerate under this Skill rather than patching legacy shape.

1. Reinspect the complete Repository Snapshot Set and reconsider every existing page against current
   source.
2. Keep only concept pages that remain useful **and** can meet the OKF page-format contract. Correct
   stale facts and citations; add concept pages for newly important reader questions; remove or merge
   pages whose purpose disappeared. Bump each edited page’s `timestamp`.
3. Re-align retained pages: required frontmatter (`type`, `title`, `description`, `timestamp`),
   relative concept edges, source evidence only under `# Citations`, no agent-written `index.md` /
   `log.md`. Repair navigation after every page-set change; ensure no old statement survives merely
   because its source file was untouched.
4. Proceed to the review reference after the whole Wiki reflects the current snapshot and contract.

Refresh is complete when every retained concept page still earns its place, all current important
ideas are navigable from **`overview.md`**, the tree is OKF-compliant, and no content depends on the
previous publication being correct.
